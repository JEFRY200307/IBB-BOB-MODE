## Template Usage Notes

**When to Create an ADR:**
- Making significant architectural decisions
- Choosing between multiple technical approaches
- Establishing patterns or standards
- Making decisions that affect multiple teams
- Decisions that are hard to reverse

**How to Use This Template:**
1. Copy this template to `docs/adr/ADR-XXX-short-title.md`
2. Replace XXX with the next sequential number
3. Fill in all sections thoroughly
4. Get review from relevant stakeholders
5. Update status as decision progresses
6. Reference this ADR in code and documentation

**ADR Status Definitions:**
- **Proposed:** Decision is under consideration
- **Accepted:** Decision has been approved and is being implemented
- **Deprecated:** Decision is no longer recommended but may still be in use
- **Superseded:** Decision has been replaced by a newer ADR
